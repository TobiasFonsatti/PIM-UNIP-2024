# 🌱 Semeando Saúde – Hortifrúti

Projeto acadêmico desenvolvido no curso de Análise e Desenvolvimento de Sistemas com foco na criação de uma solução tecnológica para modernizar a experiência de compra em hortifrútis, aliando praticidade, eficiência e sustentabilidade.

---

## 📖 Sobre o projeto

O **Semeando Saúde** é uma proposta de negócio que integra um hortifrúti físico com soluções digitais para melhorar a experiência do cliente e otimizar a gestão do estabelecimento.

O projeto surge a partir de um problema real:  
filas demoradas, baixa eficiência no atendimento e pouca integração entre tecnologia e o setor de hortifrúti.

A solução proposta combina:
- Aplicativo mobile
- Totens de autoatendimento
- Sistema de gestão integrado

---

## 🎯 Objetivos

- Melhorar a experiência de compra do cliente  
- Reduzir filas e tempo de espera  
- Automatizar processos operacionais  
- Promover consumo consciente e sustentável  
- Apoiar a tomada de decisão com base em dados  

---

## 💡 Solução proposta

O sistema **HortiTech** foi desenvolvido como uma solução modular composta por:

### 👤 Cliente
- Aplicativo para compras
- Escaneamento de produtos
- Pagamento digital
- Visualização de catálogo

### 🧾 Operadores de Caixa
- Gestão de pagamentos  
- Aplicação de descontos  
- Relatórios de vendas  

### 📦 Estoque
- Controle de entrada e saída  
- Alertas de reposição  
- Integração com fornecedores  

### 📊 Gerência
- Dashboard de vendas  
- Análise de desempenho  
- Apoio à tomada de decisão (SIG)  

---

## 📱 Funcionalidades principais

- Cadastro e login de usuários  
- Catálogo de produtos  
- Carrinho de compras  
- Checkout e pagamento  
- Feedback por avaliação  
- Autoatendimento via totem  
- Relatórios gerenciais  

---

## 🧠 Metodologias e conceitos aplicados

Durante o desenvolvimento foram aplicados conceitos fundamentais de Engenharia de Software:

- Levantamento de requisitos (funcionais e não funcionais)  
- Documentação de software  
- Modelagem UML (casos de uso)  
- Análise de negócio  
- Criação de personas  
- Prototipação de interfaces  
- Arquitetura em camadas  
- Sistema de Informação Gerencial (SIG)  
- Uso de feedback como retroalimentação  

---

## 🏗️ Arquitetura

O sistema foi pensado em uma arquitetura em camadas:

- **Front-end:** Aplicativo mobile (Android) e interfaces web  
- **Back-end:** Processamento de dados e regras de negócio  
- **Banco de dados:** Armazenamento de usuários, produtos e vendas  

---

## 🖥️ Tecnologias e ambiente

- Android (app mobile)  
- Web (navegadores modernos)  
- Windows (ambiente gerencial)  
- Servidor dedicado  
- Tablets para operação e totens  

---

## 🏪 Diferenciais do projeto

- Totem de autoatendimento com balança integrada  
- Sistema híbrido (físico + digital)  
- Integração com pagamento digital  
- Uso de dados para decisões estratégicas  
- Experiência de compra mais rápida e autônoma  

---

## 🌍 Sustentabilidade

O projeto também incorpora práticas sustentáveis:

- Agricultura vertical  
- Reaproveitamento de água (hidroponia/aquaponia)  
- Compostagem de resíduos orgânicos  
- Uso de embalagens biodegradáveis  
- Incentivo a produtores locais  

---

## 📊 Tomada de decisão com dados

O sistema utiliza:

- Feedback dos clientes  
- Dados de vendas  
- Relatórios gerenciais  

Essas informações permitem:
- Ajuste de preços  
- Identificação de produtos com baixa saída  
- Melhoria contínua do serviço  

---

## 💰 Infraestrutura

O projeto inclui planejamento completo de infraestrutura:
